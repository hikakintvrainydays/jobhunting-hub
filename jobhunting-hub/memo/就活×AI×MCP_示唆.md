## 1. MCP の本質（なぜ「履歴書」なのか）

**Ikko Eltociear Ashimine の Speaker Deck より**

```
Claude Code  = 面接官（ユーザー指示を理解し、履歴書を参照）
MCP          = 履歴書（利用可能なスキルを標準フォーマットで提供）
```

- プロトコル: JSON-RPC 2.0（既存標準を再利用）
- スコープ設定で最小権限を保証 → セキュリティ担保
- どの LLM からも同じ「型付きメソッド宣言」でスキルを読める

**意味合い**: MCP が普及すると「どのツールを使いこなせるか = 何を自動化できるか」が可視化・標準化される。就活生が「MCP を設計・運用できる」こと自体が稀少スキルになる。

## 3. 検知リスクと禁止ライン

### ❌ 明確な検知あり → 厳格禁止

| 行為                        | 検知メカニズム                                        |
| ------------------------- | ---------------------------------------------- |
| **採用マイページ・採用サイトへのAIアクセス** | IPアドレスがデータセンターIP → ログ記録 → BOT判定                |
| **Webテスト・玉手箱中のAI参照**      | フォーカス喪失・タブ切替・右クリックが技術的に記録される（arXiv:2510.18881） |
| **X(Twitter)の自動ブラウザ操作**   | webdriver検出 → アカウントBAN（即時・機械的）                 |

### ⚠️ 要注意 → 慎重に使う

| 行為 | 理由 |
|------|------|
| **Playwright で就活サイトを自動巡回** | アクセス頻度・行動パターンで検知される可能性がある |
| **AI生成ESをそのまま提出** | 採用担当が「汎用的・具体性なし・深掘り不可」で見抜く（ツールではなく人間が検知） |

### ✅ 問題なし

- IR資料・公式HP・公開ニュースをAIに分析させる
- 自分のGmail/Calendar/Notionを操作する
- X公式API（Bearer Token）でツイートを検索・取得する
- Playwright で公開採用情報ページを閲覧（ログイン不要のもの）
- ES骨子をAIで生成 → 自分の言葉に書き直して提出

## 7. 差別化の核心

```
一般就活生: ChatGPTにESを書かせる
中級者:     Claude + プロンプトで文章を磨く
上級者:     MCPパイプラインを設計し、調査〜ES〜面接まで一元管理する
→ 松井大はここ
```

**落とし穴**: AIが書いた構造のまま出す / 深掘りに答えられない / システムより実績の中身が薄い

## 8. MCP セキュリティ注意点

- **Tool Poisoning** (arXiv:2506.13538): 悪意のあるMCPサーバーがClaudeにプロンプトインジェクション攻撃を仕込む
- **対策**: 公式・著名リポジトリのみ使用。野良npmパッケージは追加しない
- **Google Drive MCP公式リポジトリはアーカイブ済み** → 代替はClaude.ai Google Workspaceコネクタ
