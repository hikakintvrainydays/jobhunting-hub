# /qa — 想定問答 生成・管理・練習コマンド

引数: $ARGUMENTS

## 操作一覧

```
/qa fill <カテゴリ or Q番号 or F番号>  → 下書き生成 → fact-checker照合 → 保存
/qa review <Q番号 or F番号>            → 既存回答を再添削・採点・改善版を出力
/qa drill <Q番号 or F番号>             → その問だけ interviewer が深掘り練習（60秒ルール）
/qa status                             → qa-status.md を読んで完了率・S級残数を表示
```

## 手順

### /qa fill

1. `profile/facts.md` を読む（SSOT確認）
2. `profile/story-parts.md`、`profile/axes.md`、`profile/weaknesses.md` を読む
3. 対象の質問を `interview/mitsubishi/想定問答_master.md` または `interview/mitsubishi/財務問答.md` から特定
4. **qa-writer** サブエージェントを呼び出して回答を生成
   - レジュメ形式（文章化禁止）
   - ✅実績のみ使用（⚠️は `[⚠️要確認]` タグ）
5. **fact-checker** サブエージェントで照合
   - ❌が1つでも出たら「要修正」とマークして保存
6. 対象ファイルの該当箇所の `<!-- 回答 -->` を回答テキストで置換
7. `interview/mitsubishi/qa-status.md` の状態を📝→🔄に更新
8. 生成した回答・採点・懸念点を表示

### /qa review

1. 対象質問の既存回答を読む
2. qa-writer の採点基準（5観点・計20点）で再採点
3. 改善版回答を生成（レジュメ形式厳守）
4. fact-checker で照合
5. 合格（16点以上）なら保存、ステータスを🔄維持
6. 「drill すれば✅になる」候補を提案

### /qa drill

1. 対象質問の回答を読む（下書きでも可）
2. **interviewer** サブエージェントに次の指示で起動:
   ```
   対象: [Q番号] [質問文]
   ルール:
   - その問だけに集中（複数問に進まない）
   - 回答が60秒（250字）超なら即「長い」と指摘
   - 最低2段の深掘りを行う（「なぜ?」「具体的には?」「あなたの役割は?」）
   - weaknesses.md の弱点と関係があれば突く
   - 終了後: 10点満点採点 + 「次回までの直し1つ」
   ```
3. drill 完了後、ステータスを🔄→✅に更新（松井が「良い」と言った場合のみ）

### /qa status

1. `interview/mitsubishi/qa-status.md` を読む
2. 優先度別の完了率を表示
3. 「今すぐ埋めるべきS級」を最大3問提示
4. 次の推奨アクションを1行で

## 鉄則（全操作共通）

- 提出物（ES・面接確定版）に使えるのは ✅ のみ
- ⚠️情報は `[⚠️要確認]` タグ付きで出力し、提出前に証拠整備を促す
- 大和の実績と松井の実績を混同しない（「自分の担当範囲は何か」を常に確認）
- 回答に空語（「成長できる環境」「グローバルに活躍」等）が出たら即削除
