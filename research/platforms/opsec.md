## なにがバレると困るか？

| リスク | バレる経路 | 影響 |
|--------|----------|------|
| AI生成ESの提出 | AI検知ツール（Turnitin系）・採用担当の直感 | ESで落とされる |
| AIが採用サイトにアクセス | IPアドレスログ（データセンターIP） | アカウント停止・選考外れ |
| 就活専用アカウントと本人の紐付け | フォロワー重複・投稿内容 | 企業が本人特定→印象悪化 |
| 筆記試験での不正（カンニング等） | 試験システムのログ | 即アウト |

## 実装ルール詳細

### ESのAI痕跡を消すチェックリスト

提出前に全項目確認。

```
AIバレ防止チェック:
□ 1. Claude で生成 → テキストエディタ（メモ帳 or VSCode）に貼付 → コピー → フォームへ
       （リッチテキスト・HTML属性が消える）
□ 2. 文末に最低2か所、自分の口語表現を混ぜた
       例: 「だと感じています」→「正直、そう思っています」
□ 3. 「〜することができました」「〜を通じて」系の定型句を1か所以上変えた
□ 4. 段落構成を崩した（起承転結→エピソード先出しに変える等）
□ 5. 数字は全てfacts.mdで確認済み（AI が架空数字を作らないよう）
□ 6. 読み上げてみて「自分の声」に聞こえるか確認
□ 7. 文字数チェック（制限の90〜98%）
```

### 採用サイトアクセスの安全な手順

```
✅ OK:
- 自分のPC or スマホ → 自宅Wi-Fi or モバイルデータ → ブラウザ(Chrome/Safari)
- 採用ページの情報をコピー → Claudeに貼付して分析
- スクショをClaude に読ませる

❌ NG:
- Claude に採用ページURLを渡して「アクセスして情報を取得して」と頼む
- WebFetch ツールで就活サイトのURLをフェッチさせる
- VPN接続中にマイページログイン（不審なIPとして記録される可能性）
```

### Xアカウントと本人の分離

```
分離レベル: 「もし採用担当が見ても問題ない」状態を保つ

メール:  shukatsu専用Gmail（本名不使用）

電話:    サブSIM推奨（ない場合は一時SMS認証サービス）
ログイン: 本アカウントとは別ブラウザorシークレットモードで使う
投稿NG: 企業・選考・面接への愚痴、大学特定できる情報、写真（GPS情報含む）
```

## 筆記試験での注意点（GABや玉手箱）

### 自宅受験（玉手箱・Web-GAB）
- カメラONの場合は部屋を整頓（カメラがOFFでも万一に備える）
- 電話・LINEの通知はオフ
- 推奨所要時間の**80〜100%**を使う（明らかに早すぎると不正疑われる）
- 途中で画面録画・キャプチャしない（利用規約違反）
- **参考書・過去問で事前対策してから受験**（一発勝負にしない）

### テストセンター（C-GAB会場など）
- 今フェーズは極力回避する
- 万一受験必要な場合: 事前にオンライン模擬問題で感覚掴む
- 受験後は「何点取れたか推定」をメモ → 同じスコアを複数社に使い回せる（C-GABは成績使い回し不可だがスコア感覚は持てる）

## Claude を安全に使うガイドライン

```
✅ ClaudeにやらせてOK:
- コピーした体験談・ES例文の分析
- 自分で書いたESの添削・改善案出し
- 企業情報のまとめ（ペーストした情報から）
- 面接練習（/drill）
- スケジュール管理・タスク整理
- テスト対策（GABや玉手箱の問題演習支援）

❌ Claudeにやらせてはいけないこと:
- マイナビ・リクナビ・ONE CAREERへのアクセス
- 採用マイページのURL → フェッチ
- 「〇〇社のES例文を検索して持ってきて」
- 「企業の採用ページから筆記形式を調べて」→ 自分で調べてコピペする
```

## 緊急時（何かバレたかも？と思ったとき）

1. そのままにする（過剰反応で更なる不審な行動をしない）
2. 採用担当への言い訳は不要（問い合わせが来るまで黙認）
3. ES再提出・内容変更は申し出ない（余計に目立つ）
4. 次の選考フェーズで挽回に集中する
