# JOB-HUNTING-HUB 要件定義書 v1.0
作成日: 2026-06-11 ／ 作成: Claude（このプロジェクトの全文脈に基づく）

## 1. 背景と目的
**ミッション**: 松井大が28卒就活で第一志望群（五大総合商社／ベンチマーク=三井物産・三菱商事）から内定を獲得する。

**本ハブの役割**: 「企業を知る」工程は既存の company-analyzer が担う。本ハブは「自分を売る」工程の全て——実績管理・ES・面接・選考管理・日次運用——を一元化し、両者を接続する。

**解決する課題（このプロジェクトで繰り返し顕在化した問題）**:
1. **実績の確度混在** — TOEIC 790/825/935 の揺れ、ToruTour「停止」vs「継続」、AVR JAPAN「構想」vs「実績」。放置すると誇張→面接の深掘りで破綻→リスクチェック落ち。
2. **志望先の拡散** — 商社・海運・食品・メーカー・IT・コンサルを同時深掘りすると全部薄くなる。
3. **締切・選考状況の属人管理** — 早期選考は情報戦。漏れ=機会損失。
4. **ESと想定問答が都度生成で蓄積されない** — 添削の学びが次に活きない。
5. **弱点への返しが未整備** — GPA2.3／海外3ヶ月／事業停止／大和との役割分担。

## 2. スコープ
- **In**: 実績SSOT管理／ファクトチェック／ES生成・添削・保管／模擬面接／選考パイプライン／日次・週次運用／弱点対策／26卒先輩ベンチマーク参照／company-analyzer連携
- **Out**: 企業分析そのもの（company-analyzerに委譲）／応募の自動送信・メール送信等の外部操作（行わない）

## 3. ユーザー・環境
- ユーザー: 松井大（単独利用）
- 環境: Claude Code。外部API非依存。全データはリポジトリ内の md / csv（Git管理可能）。

## 4. 機能要件
| ID | 機能 | 実装 |
|---|---|---|
| F-1 | 実績SSOT: 全実績を ✅確定/⚠️要確認/💭構想/⏸停止 の確度タグ+証拠パスで一元管理 | `profile/facts.md` |
| F-2 | ファクトチェック: 成果物内の数字・固有名詞をF-1と照合し、証明不能を提出前に検出 | `agents/fact-checker` |
| F-3 | ES工場: 設問意図分解→部品選定→生成→5観点採点の鬼添削→証拠照合→保管 | `/es` + `agents/es-writer` + `agents/es-reviewer` |
| F-4 | 面接道場: 商社面接官ペルソナで深掘り模擬面接（なぜ5連発・想定外質問・講評） | `/drill` + `agents/interviewer` |
| F-5 | 選考パイプライン: 企業×優先度×ステータス×締切の一元管理（S5/A8/B7=20社上限） | `/pipeline` + `pipeline/companies.csv` |
| F-6 | 日次・週次運用: 締切4週間スキャン→今日の3タスク／週次リソース再配分 | `/today` `/weekly` + `agents/strategist` |
| F-7 | company-analyzer連携: INTERVIEW_PACK.md を読み込みES・問答素材に変換 | `/analyze` |
| F-8 | ベンチマーク: 26卒先輩20名のプロフィールと照合し「国際派テンプレ」との差別化を検証 | `benchmarks/` |

## 5. 非機能要件
- **正直性**: facts.md にない数字・主張を成果物に使わない。⚠️情報は出力にも⚠️を継承。提出物（ES等）は✅のみ。
- **鮮度**: 各コマンドは facts / pipeline の最終更新日を確認し、7日超なら更新を促す。
- **個人情報**: パスワード・マイページ認証情報の平文保存禁止（サービス名のみ可）。
- **可搬性**: 全データはテキスト。どの環境のClaude Codeでも動く。

## 6. データ設計（主要ファイルのスキーマ）
- `profile/facts.md` — 実績台帳。項目=実績／確度タグ／数字／証拠パス／注意。
- `profile/story-parts.md` — ガクチカ部品庫。「違和感→仮説→行動→調整→結果→学び」+企業別変換表+深掘り耐性メモ。
- `pipeline/companies.csv` — 優先度,企業名,業界,軸,ステータス,次アクション,ES締切,Webテスト,面接予定,メモ
- `es/submitted/<企業>/<日付>_<設問>.md` — 提出版アーカイブ（採点結果付き）。
- `interview/qa-bank.md` — 想定問答バンク（汎用）。企業別は `interview/<企業>/`。
- `evidence/index.md` — 証拠台帳（実績↔証拠ファイル/URLの対応）。

## 7. 運用フロー
- **毎朝（5分）**: `/today` → 今日の3タスク
- **ES提出前（必須）**: `/es <企業> <設問> <字数>` → fact-checker通過がゲート
- **面接前**: `/analyze <企業>` → `/drill <企業> <段階>`
- **毎週日曜（15分）**: `/weekly` → リソース再配分・facts棚卸し

## 9. リスクと対策
- **誇張リスク** → F-2で機械的に遮断（❌が1つでもあれば提出不可）。
- **拡散リスク** → companies.csv の S/A/B 枠上限。新規追加時は必ず1社落とす。
- **大和依存リスク** → facts.md は「自分の担当範囲」明記がない実績の登録を不可とする。
- **鮮度リスク** → 更新日チェックを全コマンドの起動時ルーチンに組込み。
