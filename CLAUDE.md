# CLAUDE.md — job-hunting-hub 全体の常時指示

このリポジトリで作業するすべての Claude Code セッションは、以下を常に守ること。
（`company-analyzer/` 内の作法は、当該フォルダの `CLAUDE.md` を正とする。本ファイルはハブ全体の共通ルール。）

## 1. 変更を必ずログに残す（最重要）

作業のたびに **`CLAUDE_log.md` の末尾へ追記** する。1エントリは数行でよい：

- **日時**（JST）
- **触ったファイル**（追加・変更・削除）
- **変更概要**（何をしたか）
- **判断理由**（なぜそうしたか）
- **次アクション**（次に何をすべきか／保留事項）

目的は「変更を常に追えること」。GitHub のコミット差分と `CLAUDE_log.md` の二本立てで、意思決定の流れを後から再現できる状態を保つ。

## 2. 事実と推論を分ける／数字を創作しない

- 資料にない数字・実績・固有名を **創作しない**。
- 事実（資料に明記）・本人申告・推定・未検証情報を明確に区別して書く。
- 特に TOEICスコア・GPA・受賞・海外滞在期間・事業実績・売上/利益・提携店舗数などは、
  証明可能な情報だけを企業提出物・面接回答に使う（`profile/matsui_dai_knowledge_file_v1.md` の取り扱い方針に従う）。
- これは `company-analyzer` の鉄則だが、ハブ全体に適用する。

## 3. 資料はそのまま扱う

- `profile/` `strategy/` `seminar/` の一次資料は **原則そのままコピー**。中身を改変・要約で上書きしない。
- 加工が必要なときは新しいファイルとして作り、元資料は残す。

## 4. 第三者の個人情報の取り扱い

- 本リポジトリは非公開。ゼミ同期のプロフィール（`profile/2026卒_就活生プロフィール_…csv`）や
  LINEログ（`seminar/LINE_…txt`）は第三者の個人情報を含む。
- 公開リポジトリへの転載・外部サービスへの不用意な送信をしない。

## 5. コミット規約

- コミットは小さく。メッセージは `分類: 要約` 形式（例 `feat: company-analyzer移植`, `docs: ログ運用追記`）。
- **force push / rebase 後の上書きは禁止**（履歴を改変しない）。
- 破壊的操作（既存リポジトリ削除・履歴改変）は実行前に必ず人間に確認する。

## 6. company-analyzer の使い方

- 企業分析の依頼は `分析: <企業名>` で起動する。作法は `company-analyzer/CLAUDE.md` が正。
- 本人の人物像は `profile/matsui_dai_knowledge_file_v1.md` を最重要ソースとする。
